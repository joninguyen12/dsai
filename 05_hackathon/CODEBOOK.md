# Codebook: Seattle Congestion Insights App

A **codebook** documents the data, variables, and conventions used in a project so that others (or future you) can understand and reuse the data and API without reading the source code. This codebook covers the database schema, REST API, and dashboard inputs/outputs for the Seattle Congestion Insights app.

---

## 1. Overview

- **Purpose:** Surface congestion patterns for key Seattle-area freeway locations using synthetic time-series data.
- **Pipeline:** Supabase (PostgreSQL) → FastAPI REST API → Shiny dashboard → optional AI summary (Ollama Cloud).
- **Data:** Synthetic congestion readings generated at 15-minute intervals for the last 7 days, across 18 predefined intersections.

---

## 2. Data Sources & Collection

| Item | Description |
|------|-------------|
| **Database** | Supabase (PostgreSQL), accessed via PostgREST. |
| **Locations** | Seeded once via `seed_locations.py` (18 fixed intersections). |
| **Readings** | Generated by `generate_congestion_data.py`: 15-min intervals, 7 days back from run time, one row per location per time step. |
| **Data type** | Synthetic (not live sensors). `data_source` in readings is `"synthetic_v1"`. |

## 3. Database Tables & Variables

### 3.1 Table: `locations`

Freeway intersection / segment reference list. One row per location.

| Variable | Type | Description | Example / Notes |
|----------|------|-------------|-----------------|
| `id` | UUID | Primary key (auto-generated by Supabase). | `a1b2c3d4-...` |
| `name` | text | Human-readable intersection name. | `"I-5 & I-90 Interchange"` |
| `freeway` | text | Highway identifier. | `"I-5"`, `"I-90"`, `"I-405"`, `"SR-99"`, `"SR-520"` |
| `cross_street` | text | Cross street or interchange. | `"I-90"`, `"James St"` |
| `direction` | text | Travel direction(s). | `"NB/SB"`, `"EB/WB"` |
| `city` | text | City name. | `"Seattle"`, `"Bellevue"`, `"Mercer Island"` |
| `neighborhood` | text | Neighborhood or area. | `"Downtown"`, `"Montlake"` |
| `latitude` | float | WGS84 latitude. | `47.6097` |
| `longitude` | float | WGS84 longitude. | `-122.3331` |
| `is_active` | boolean | Whether location is included in seeding/queries. | `true` |

**Seed script:** `seed_locations.py`. Locations are fixed; no automatic updates.

---

### 3.2 Table: `congestion_readings`

Time-stamped congestion and traffic metrics per location.

| Variable | Type | Description | Valid range / Notes |
|----------|------|-------------|---------------------|
| `id` | UUID | Primary key (auto-generated). | — |
| `location_id` | UUID | Foreign key to `locations.id`. | Must exist in `locations`. |
| `observed_at` | timestamptz | Time of the reading (UTC). | ISO 8601, e.g. `2026-03-09T12:00:00Z` |
| `congestion_index` | integer | Congestion severity (0 = free flow, 100 = severe). | 0–100 |
| `speed_mph` | float | Average speed in miles per hour. | Typically ~10–65; derived from congestion in synthetic data. |
| `vehicle_count` | integer | Vehicle count in the interval (synthetic). | Positive integer. |
| `travel_time_index` | float | Ratio of actual travel time to free-flow time (>1 = slower). | e.g. 1.0–2.2 |
| `is_incident` | boolean | Whether the reading is tagged as an incident. | `true` / `false` (synthetic data uses `false`). |
| `data_source` | text | Origin of the record. | `"synthetic_v1"` for generated data. |

**Generation:** `generate_congestion_data.py`; interval = 15 minutes; window = last 7 days from run; 24-hour baseline pattern plus location bias and random noise.

## 4. REST API Reference

Base URL (local): `http://127.0.0.1:8000`  
Base URL (Posit Connect): set via `CONGESTION_API_URL` for the Shiny app.

All times in query params and responses are ISO 8601 UTC (e.g. `2026-03-09T04:00:00Z`).

### 4.1 Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Service info and list of endpoints. |
| GET | `/locations` | List locations (optional filters). |
| GET | `/locations/{location_id}` | Single location by ID. |
| GET | `/readings` | Paginated congestion readings (filters: location, time window, severity). |
| GET | `/readings/top` | Top N most congested readings in a time window. |
| GET | `/readings/summary` | Aggregate stats (count, avg congestion, avg speed) for a time window. |

### 4.2 Query Parameters

**GET /locations**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `freeway` | string | No | Filter by freeway (e.g. `I-5`). |
| `city` | string | No | Filter by city. |

**GET /readings**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `location_id` | UUID | No | Filter by location. |
| `from` | ISO datetime | No | Start of time window. |
| `to` | ISO datetime | No | End of time window. |
| `min_congestion` | int | No | Minimum `congestion_index` (0–100). |
| `max_congestion` | int | No | Maximum `congestion_index` (0–100). |
| `limit` | int | No | Max rows (default 500, max 2000). |
| `offset` | int | No | Pagination offset (default 0). |

**GET /readings/top**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | int | No | Number of top readings (default 10, max 50). |
| `from` | ISO datetime | No | Start of window; default = 1 hour before `to`. |
| `to` | ISO datetime | No | End of window; default = now. |
| `location_id` | UUID | No | Restrict to one location. |
| `min_congestion` | int | No | Minimum `congestion_index` (0–100). |

**GET /readings/summary**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | ISO datetime | No | Start of window; default = old date. |
| `to` | ISO datetime | No | End of window; default = now. |

### 4.3 Response Shapes

- **`/locations`**, **`/locations/{id}`:** Array or object with `locations` table fields (e.g. `id`, `name`, `freeway`, `city`, `neighborhood`, `latitude`, `longitude`, …).
- **`/readings`**, **`/readings/top`:** Array of objects with at least: `id`, `location_id`, `observed_at`, `congestion_index`, `speed_mph`, and optionally `vehicle_count`, `travel_time_index`, `is_incident`.
- **`/readings/summary`:** `{ "from", "to", "count", "avg_congestion", "avg_speed_mph" }` (nulls if no data).

## 5. Dashboard (Shiny App) Inputs & Outputs

**Entry point:** `app.py` (Shiny for Python).

### 5.1 Environment / Configuration

| Variable | Purpose |
|----------|---------|
| `CONGESTION_API_URL` or `API_URL` | Base URL of the FastAPI service. Default: `http://127.0.0.1:8000`. On Posit Connect, set to the deployed API content URL. |

### 5.2 User Inputs (Sidebar)

| Input ID | Type | Description | Values / Notes |
|----------|------|-------------|----------------|
| `time_preset` | select | Time range preset. | `"24h"`, `"7d"`, `"custom"` |
| `date_range` | date range | From–To dates (only when preset = custom). | Bounded by last 7 days of seeded data. |
| `location_id` | select | Location filter. | `""` = All locations; otherwise UUID from API. |
| `top_n` | numeric | Number of rows in Traffic Hotspots table. | 1–50 (default 10). |

### 5.3 Outputs (Tabs)

| Tab | Content | Data source |
|-----|---------|-------------|
| **Traffic Hotspots** | Table: rank, date/time, intersection, city, neighborhood, congestion %, speed. | `GET /readings/top` + `GET /locations` for labels. |
| **Explore Trends** | Plotly line chart: congestion index (hourly average) vs. time, by location. | `GET /readings` (paginated) + locations. |
| **AI Summary** | Bullet-point summary of congestion in the selected window. | Same readings + `ai_summary.summarize_congestion_data()` (Ollama Cloud). |

### 5.4 Internal Conventions

- **Time window:** Converted to `from` / `to` ISO UTC and passed to API.
- **Trends chart:** Readings are downsampled to hourly means; multi-location shows one line per location.
- **Pagination:** Dashboard fetches up to `PAGE_SIZE * MAX_PAGES` readings (see `app.py`); a disclaimer is shown when the data cap is reached.

## 6. Codes & Conventions

| Item | Convention |
|------|-------------|
| **Time zone** | All timestamps UTC (ISO 8601 with `Z` or `+00:00`). |
| **Congestion index** | 0 = free flow, 100 = maximum congestion; integer. |
| **Freeway codes** | `I-5`, `I-90`, `I-405`, `SR-99`, `SR-520` (Washington State). |
| **Direction codes** | `NB`/`SB` (northbound/southbound), `EB`/`WB` (eastbound/westbound). |
| **Data source tag** | `synthetic_v1` for script-generated readings. |
